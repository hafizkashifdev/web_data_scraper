Frequently Asked Questions

What does the developer portal cover in terms of APIs?

This developer portal is published as per the open banking regulation mandate & the APIs in line with the specification. It covers AISP & PISP functionalities.

Is there separate API for corporate & retail banking customers?

No. Its the same APIs for both retail & corporate customers. The way customers login & authorise may be different but the APIs itself are based on the open banking specifications.

How is consent gathered for AISP flow?

Our API specification only supports Bank Offered Consent, which means the AISP doesn't explicitly specify the list of customer accounts of customers during Consent Request. But during consent capture, bank will display list of all available bank accounts that customer can choose to share Account Information.

Do Developer/TPP need to be authorised on NCA register?

We do expect developers to be licensed. However this is flexible in test or sandbox systems. We use Banfico's directory services to connect to 31 NCAs.

Does Developer need eIDAS certificate for secure network connections & application data signing?

Yes, this is needed in PRODUCTION, but not needed for testing in Sandbox.

Do I get test bank customer accounts to perform AISP/PISP flows?

Yes. Actually you could however register as a bank customer on our reference bank application. These customers can create their own list of accounts, balances & transactions.

How can consent be revoked?

Bank customers can login into consent module application & manage their consent. Revoke for relevant AISP grants.

Are there any other API specification supported?

No. We currently only support Open Banking UK API specification.

Can developers create more apps & subscribe to same AISP/PISP API product?

Yes. In production mode, developers can only create APP that has FCA license associated with it. However in test-mode, developers can create any number of AISP or PISP apps.

FAPI FAQs

What changes has Banfico made to its FAPI-defined endpoints?

Banfico has updated its FAPI-defined endpoints (/token, /authorize) to comply with the OIDF FAPI 1.0 – Advanced specification. These changes include new validation requirements for fields defined in FAPI 1.0 – Advanced, such as the ‘nbf’ claim.

What is the impact of the update on my integration?

The updates apply immediately, but there will be a period of backwards compatibility with FAPI 1.0 Implementers Draft 2. TPPs should expect to adapt to the new specifications as Banfico's sandbox environment is updated.

When do these changes take effect?

The updates apply immediately, but there will be a period of backwards compatibility with FAPI 1.0 Implementers Draft 2. TPPs should expect to adapt to the new specifications as Banfico's sandbox environment is updated.

Will there be any backward compatibility?

Yes, Banfico will maintain backwards compatibility with FAPI 1.0 Implementers Draft 2 for a transitional period. During this time, TPPs can continue to use existing implementations, but must update their systems to support the FAPI 1.0 – Advanced specification.

What should TPPs do to ensure compatibility with Banfico’s updates?

TPPs should:

  • Verify they are supporting UTF-8 as the default character encoding for API responses, as required by the Read/Write standard.
  • Ensure they are providing the appropriate fields (e.g., the ‘nbf’ claim) as specified in FAPI 1.0 – Advanced.
  • Prepare for eventual mandatory compliance with FAPI 1.0 Final once backwards compatibility is no longer supported.

Will there be any disruption to service during this transition?

There should be no service disruption if your application adheres to the updated FAPI 1.0 – Advanced specification. TPPs should update their systems accordingly to avoid any potential issues.

When does backwards compatibility end?

Backwards compatibility with FAPI 1.0 Implementers Draft 2 will end after a set period. TPPs will need to fully align with FAPI 1.0 Final, including mandatory fields like the ‘nbf’ claim, by the time this period concludes.

What happens if I don’t comply with the updated requirements?

If your implementation does not comply with the new FAPI 1.0 – Advanced specification after the transition period, Banfico may reject requests that are missing mandatory fields (e.g., ‘nbf’ claim) or fail to support the UTF-8 encoding.

Where can I get more information or support for these changes?

For additional information, please refer to Banfico’s developer documentation or reach out to our support team. Our technical support team is available to assist with the integration of the updated FAPI specifications and to ensure smooth compliance.
